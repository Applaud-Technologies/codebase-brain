# Codebase Brain Local Artifacts

This directory contains local analysis artifacts generated by Codebase Brain.

## Directory Structure

```
.codebase-brain/
├── README.md           # This file
└── runs/               # Analysis run artifacts
    ├── job_<id>/       # Individual run directory
    │   ├── result.json # Full analysis results (JSON)
    │   └── report.md   # Human-readable health report
    └── cleanup-log.jsonl  # Audit log of deleted runs
```

## Retention Policy

By default:
- **Latest 25 completed runs** are kept
- **Completed runs older than 30 days** are deleted
- **Failed/timed-out runs older than 7 days** are deleted
- **Pinned runs** are never automatically deleted
- **Active runs** (queued, running, finalizing) are never deleted

## Configuration

Retention settings can be customized in `codebase-brain.config.json`:

```json
{
  "retention": {
    "keep_latest": 25,
    "max_age_days": 30,
    "failed_max_age_days": 7
  }
}
```

## Commands

- `analyze_repository` - Start a new analysis
- `get_analysis_result` - Get status/results of an analysis
- `list_analysis_runs` - List all runs for this repository
- `cleanup_analysis_runs` - Manually trigger cleanup (use `dry_run: true` first)

## Git

The `runs/` directory is ignored by git. Only this README and configuration files should be committed.
