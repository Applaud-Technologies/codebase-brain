The Problem

AI doesn't know what you've already built

Ask an AI to add email validation and it writes a new method—even if you already have three validators in your codebase. Every AI session compounds technical debt.

Without Codebase Brain

You: Add email validation

AI: I'll create a new method...

Writing ValidateEmail()

Result: 4th email validator in codebase

With Codebase Brain

You: Add email validation

AI: Searching existing code...

Found: EmailValidator.IsValid()

Match: 95% similar

AI: Recommend reusing existing method

MCP Tools

Four tools that enforce the workflow

find_existing_behavior

Semantic search by description. "Calculate shipping cost" finds ComputeShippingCost() even when names differ.

find_symbols

Pattern-based symbol search. Find all methods matching *Shipping* across the codebase.

detect_duplicates

Check if proposed code duplicates existing functionality before writing it.

pre_write_guard

Blocks code generation unless search evidence is provided. The enforcement mechanism.

🧠 Semantic

Vector embeddings understand what code does, not just what it's named. Finds similar behavior across different naming conventions.

🔗 Structural

Code graph tracks calls, dependencies, and relationships. Understands how code connects and what depends on what.

📝 Lexical

Fast exact-match text search. Pattern matching and regex across your entire codebase in milliseconds.

Quick Start

Up and running in 2 minutes

# Clone and start the services
git clone https://github.com/Applaud-Technologies/codebase-brain.git
cd codebase-brain
docker compose up -d

# Connect Claude Code to Codebase Brain
claude mcp add codebase-brain --type http --url http://localhost:3100/mcp

# Verify connection
claude mcp list
codebase-brain: http://localhost:3100/mcp (HTTP) - ✓ Connected
